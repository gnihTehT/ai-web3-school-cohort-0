# Handbook 反馈流程

> 当学习过程中发现 Handbook 内容有问题、过时、不清晰或缺失时，按此流程记录反馈。

## 触发条件

- 页面 404 / 链接失效
- 概念解释有误或令人困惑
- 代码示例跑不通
- 缺少重要内容
- 翻译/语言问题

## 反馈草稿格式

```markdown
## 反馈

### 章节
[Handbook 章节名 / URL]

### 问题类型
[链接失效 / 概念错误 / 表述不清 / 内容缺失 / 其他]

### 具体描述
[遇到的问题或建议]

### 建议修改
[如果有改进建议，写在这里]
```

## 流程

1. 学习时发现 Handbook 有问题 → Lead dog 指出
2. Thound 整理成反馈草稿（以上格式）
3. Lead dog 审核确认
4. ✅ 提交到 WCB 平台或对应反馈渠道（经确认后执行）
5. 记录到 `memory/feedback-drafts/` 存档
